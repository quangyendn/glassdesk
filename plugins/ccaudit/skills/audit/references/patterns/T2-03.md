# T2-03 — CLAUDE.md scope split

## Why it matters
User-level CLAUDE.md (`~/.claude/CLAUDE.md`) should hold cross-project preferences. Project-level (`.claude/CLAUDE.md`) should hold repo-specific rules. Mixing them means project-specific tokens load in every session.

## Detection
Informational check: are both user and project layers present? User layer = `~/.claude/CLAUDE.md`. Project layer = `./CLAUDE.md` OR `./.claude/CLAUDE.md` (either counts). Emits 1 if only one level exists.

## Target
Both user-level (`~/.claude/CLAUDE.md`) and project-level (`./CLAUDE.md` or `./.claude/CLAUDE.md`) present, with appropriate scope separation.
