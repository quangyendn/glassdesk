# T2-09 — Linter-handled formatting rules

## Why it matters
If a linter config (eslint, prettier, ruff, rubocop, editorconfig) already enforces formatting, repeating those rules in CLAUDE.md creates conflicting sources of truth and wastes tokens.

## Detection
Scans every existing CLAUDE.md memory layer (see helper `_claude_md_files`). Emits 1 only if BOTH a formatting keyword appears in any CLAUDE.md layer AND a linter config file exists (conflict detected).

## Target
0 conflicts — let the linter own formatting; remove duplicate rules from CLAUDE.md.
